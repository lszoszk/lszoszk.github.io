What is Natural Language Processing?
nlp_intro.md

DEFINITION:

Natural Language Processing (NLP) is a field of artificial intelligence that enables computers to understand, interpret, and generate human language. It's the bridge between human communication and machine computation.

Key NLP tasks include:
‚Ä¢ Search: Find relevant documents from large collections
‚Ä¢ Classification: Categorize documents by topic, sentiment, or type
‚Ä¢ Extraction: Pull out specific information (names, dates, concepts)
‚Ä¢ Analysis: Discover patterns and trends in text data
‚Ä¢ Summarization: Condense long documents into key points

What is Vibe-Coding?
vibe_coding_intro.md

DEFINITION:

Vibe-coding is programming through natural language conversation with AI assistants. Instead of writing code syntax, you describe what you want to build in plain English (or Italian!).

THE PARADIGM SHIFT:

  • ‚úì Focus on ideas and outcomes, not syntax
  • ‚úì Iterate rapidly through conversation
  • ‚úì Learn "on demand" - depth-wise, not breadth-wise
  • ‚úì Accessible to domain experts without CS degrees

‚öñÔ∏è WHY HUMAN RIGHTS NEEDS NLP & AI

Human rights work is fundamentally about text: treaties, reports, legal documents, testimonies. Even basic NLP techniques can be gamechangers for human rights work:

  • Transform collection of documents into searchable databases
  • Find patterns invisible to human reading (trends across decades of reports)
  • Quantify arguments with data (e.g., "Internet access" mentioned 27 times in 2024 reports - see example)
  • Build tools like General Comments Database
  • Democratize access to legal information

üîß DAY 1: Basics of Natural Language Processing

  • Setup: Introduction to Google Colab
    Google Colab is a free cloud-based Jupyter notebook environment. No installation needed - just open your browser, log in with Gmail, and start coding! Perfect for beginners and collaborative work. We'll create our first notebook and learn basic navigation.
  • Data Transformation: XLSX to JSON
    We'll convert human-readable spreadsheets into machine-readable JSON format. Using the CCPR dataset as our example, you'll see how structured data becomes programmable. This is the foundation for all data analysis work.

    üìä Dataset links:
    ‚Ä¢ XLSX Dataset
    ‚Ä¢ JSON Dataset
  • Text Preprocessing:: Understanding text as data, binary representation
    Computers don't "read" text - they process numbers. Every letter is a number in binary. 
    Example: The word "law" in binary
L = 01001100 (ASCII: 76)
a = 01100001 (ASCII: 97)
w = 01110111 (ASCII: 119)

"law" = 01001100 01100001 01110111
    Understanding this helps you grasp how computers process human rights documents! Before we can use the data, we need to preprocess it:
    • Cleaning text (removing special characters, formatting, etc.)
    • Tokenization (splitting text into words)
    • Normalization (lowercase, stemming - reducing words to their root form, lemmatization - reducing words to their base form)
    • Stop word removal (removing common words like "the", "and", "is")
  • Search: Search & retrieval
    Build your first search engine! Implement keyword search, Boolean operators (AND/OR), and fuzzy search. You'll transform documents into a searchable database.
  • Frequency Analysis
    Discover patterns in text by counting words (term frequency) and word pairs (bigrams). This simple technique reveals what topics/concepts dominate human rights discussions and how concepts connect. For instance, "human" + "rights" appearing together tells us more than counting them separately.
  • Text Classification: Dictionary methods, topic modeling, LLMs
    Three approaches to categorize documents:
    ‚Ä¢ Dictionary: Using predefined word lists (fast, transparent)
    ‚Ä¢ Topic modeling: Let the algorithm discover themes (unsupervised)
    ‚Ä¢ LLMs: Using AI for nuanced classification (powerful but opaque)

    We'll run discuss their trade-offs for human rights research. 
        "Children": [" child", "girl", "boy", "juvenile"]
    "Persons deprived of their liberty": ["prison", incarcerated, "detained", "detention", "custody", "confinement"],

    See full dictionary
  • Export: Saving results to Excel
    Transform your analysis into shareable reports. We'll export data frames to Excel and prepare findings for non-technical colleagues. Making results accessible is as important as the analysis itself!

üöÄ DAY 2: Building Your Own App - Document Database

  • Design: Focus on needs, prototype in AI assistant (using canvas function)
    Start with the problem, not the technology. What do human rights defenders actually need? We'll use Claude's or Google's tools to rapidly prototype interfaces. Remember: the best app is the one that gets used, not the most sophisticated one.
  • Data sources: Existing datasets, APIs, web scraping
    Three ways to get data:
    ‚Ä¢ Existing datasets: HuggingFace, GitHub repositories
    ‚Ä¢ APIs: Direct access to live data (UN databases, government portals)
    ‚Ä¢ Web scraping: Extract data from websites (ethically and legally!)
    We'll explore selected methods with human rights examples.
  • Preprocessing: websites, PDFs, old documents
    The unglamorous truth: ca. 85% of your time goes here! We'll discuss:
    ‚Ä¢ HTML extraction: Getting text from websites
    ‚Ä¢ PDF extraction: Getting text from documents
    ‚Ä¢ OCR: Converting images to text (for old documents, travaux preparatoires)

    üìÅ Illustrative pipeline: PDF ‚Üí Markdown ‚Üí JSON transformation
    See every stage of General Comments preprocessing!
  • Database features: Chunking and labeling
    Transform raw text into searchable knowledge:
    ‚Ä¢ Chunking: Split documents into meaningful segments (paragraphs, sections)
    ‚Ä¢ Labeling: Tag content (country, year, treaty body, concerned groups)
    ‚Ä¢ Metadata: Add context (hyperlinks, author, date, document type)
    Good structure makes the difference between a toy and a tool!
  • UX Design: Adjusting ideas to technical realities
    Dream big, build pragmatically. We'll learn to:
    ‚Ä¢ Simplify complex interfaces for non-technical users
    ‚Ä¢ Choose between search, browse, and filter approaches
    ‚Ä¢ Design for mobile (field workers often only have phones)
    ‚Ä¢ Balance features with performance
    Remember: A simple tool that works beats a complex one that doesn't!
  • Deployment options: local vs. server
    Local deployment (on your computer):
    ‚úÖ Easy setup, full control, no security risks
    ‚ùå No collaboration, not accessible to others

    Server deployment (online, e.g., PythonAnywhere):
    ‚úÖ Accessible worldwide, shareable, professional
    ‚ùå Requires security measures, costs money, more complex

    We'll learn how to deploy a simple app on PythonAnywhere - your first public human rights tool!
